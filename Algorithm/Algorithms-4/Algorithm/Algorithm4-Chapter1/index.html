此代码是一个简单的 “解释器”： 一个能够解释给定字符串所表达的运算并计算得到结果的程序

解释器（英语：Interpreter），又译为直译器，是一种电脑程序，能够把高级编程语言一行一行直接转译运行。解释器不会一次把整个程序转译出来，只像一位“中间人”，每次运行程序时都要先转成另一种语言再作运行，因此解释器的程序运行速度比较缓慢。它每转译一行程序叙述就立刻运行，然后再转译下一行，再运行，如此不停地进行下去

对象游离

Java 的垃圾回收（GC）回收所有无法访问的内存，在实现 pop( ) 的过程中，被弹出元素的引用仍存在于数组之中。实际上这个元素不会再被访问，但是垃圾回收器无法知道这一点，除非改引用被覆盖。要实现这一点，将弹出元素的值设为null。这将覆盖无用的引用，并且使系统可以在用例使用完被弹出元素后回收它的内存。

综述

数组被称为： 顺序存储

  • 优点： 通过索引可以直接访问任意元素
  • 缺点： 在初始化时需要知道元素的数量

链表被称为： 链式存储

  • 优点：使用的空间大小和元素数量成正比
  • 缺点：需要通过引用访问任意元素
