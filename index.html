数据仓库的特点：

  • Subject-Oriented

  • Integrated

    ![Screenshot 2022-05-05 at 12.18.45](/Users/zhangyuyang/Desktop/blog/source/_posts/itheima/hive/Screenshot 2022-05-05 at 12.18.45.png)

  • Non-Volatile：

    数据仓库的数据，反应的是一段长时间内历史数据的内容，不同时间点的快照。有大量查询操作，修改和删除操作很少

  • Time-Variant

OLAP：on-line Analytical Processing： Data Warehouse

Data Mart：面向单个部门，只涉及某个主题

![Screenshot 2022-05-05 at 12.26.27](/Users/zhangyuyang/Desktop/blog/source/_posts/itheima/hive/Screenshot 2022-05-05 at 12.26.27.png)

每个企业根据自己的业务需求可以分成不同的层次，但是最基本的分层思想：

  • 操作层（ODS）： 临时缓存数据，此层存放未经处理的原始数据至数据仓库系统，结构上与源系统保持一致，是数据仓库的数据准备区，与数据源系统进行解耦合
  • 数据仓库层（DW）：有ODS层数据加工而成，主要完成数据加工与整合。
  • 数据应用层（DA）：提供给产品和数据分析使用的数据

![Screenshot 2022-05-05 at 12.27.20](/Users/zhangyuyang/Desktop/blog/source/_posts/itheima/hive/Screenshot 2022-05-05 at 12.27.20.png)

![Screenshot 2022-05-05 at 12.28.41](/Users/zhangyuyang/Desktop/blog/source/_posts/itheima/hive/Screenshot 2022-05-05 at 12.28.41.png)

为什么要分层：

  • 清晰数据结构
  • 数据血缘追踪
  • 减少重复开发
  • 把复杂问题简单化
  • 屏蔽原始数据的异常

HIVE

hive将存储在Hadoop文件中结构化和半结构化的文件映射为一张数据库表

hive的核心是将 HQL转换为MapReduce 程序，然后将程序提交到Hadoop集群执行

Hive将文件映射成一张表，映射：映射的专业叫法叫做元数据信息

元数据：

  • table 对应着哪个 文件（位置信息）
  • table 的列对应着文件的哪个字段（顺序信息）
  • 文件字段之间的分隔符是什么

内部表外部表

内部表（Internal table）也称为Hive拥有和管理的托管表（Managed table）。默认情况下创建的表是内部表，Hive拥有该表的结构和文件，换句话说Hive完全管理（元数据和数据）的生命周期，类似于 RDBMS中的表，到删除表时，会删除数据以及表的元数据

Linux 安装vim失败

![Screenshot 2022-04-22 at 22.33.40](/Users/zhangyuyang/Desktop/blog/source/_posts/杂记/Screenshot 2022-04-22 at 22.33.40.png)

解决办法：把vim卸载了重新安装

  • sudo apt-get purge vim-common： 卸载

  • sudo apt-get update： 对软件包列表进行更新

  • sudo apt-get upgrade： 对已经安装的软件包本身进行更新

  • sudo apt-get install vim： 安装

apt-get purge 与 apt-get remove 的区别:

apt-get remove 会删除软件包而保留软件的配置文件
apt-get purge 会同时清除软件包和软件的配置文件
